## Build Your First Decentralized Application (DApp)

Learn to use Hedera network services and build your first dApp with step-by-step tutorials, or dive into the interactive [Hedera Developer Playground](https://portal.hedera.com/playground) for a hands-on coding experience.

<CardGroup cols={3}>
<Card title="Create & Deploy a Smart Contract" href="/core-concepts/smart-contracts/deploying-smart-contracts" img="/images/index/index-8.png">

Create a Solidity Smart Contract that interacts with the Hedera Token Service, bringing full programmability into your token-based application.

</Card>

<Card title="Create Fungible Tokens" href="/tutorials/token/create-and-transfer-your-first-fungible-token" img="/images/index/index-9.png">

Service to create fungible tokens that map to [ERC-20 standards](https://docs.hedera.com/guides/core-concepts/smart-contracts/supported-erc-token-standards) and scale to 10,000+ TPS.

</Card>

<Card title="Create Non-Fungible Tokens" href="/tutorials/token/create-and-transfer-your-first-nft" img="/images/index/index-10.png">

Use the Hedera Token Service to create non-fungible tokens that map to [ERC-721 standards](https://docs.hedera.com/guides/core-concepts/smart-contracts/supported-erc-token-standards) and scale to 10,000+ TPS.

</Card>

<Card title="Write Verifiable Data to Hedera" img="/images/index/index-11.png">

Use the Hedera Consensus Service as a decentralized message bus to write immutable, verifiable, and consensus timestamped data at 10k+ TPS and $0.0001 per message.

</Card>

<Card title="Hedera Developer Playground" href="https://portal.hedera.com/playground" img="/images/index/index-12.png">

Use the interactive Hedera Developer Playground to follow guided examples or experiment with Hedera SDKs in a hands-on coding environment.

## EVM-Compatible Tools

Continue utilizing familiar Ethereum development tools such as Web3.js, Truffle, Ethers, Hardhat, and Foundry to build on Hedera using the [JSON-RPC Relay](/hedera/core-concepts/smart-contracts/json-rpc-relay). As an Ethereum developer, your workflow does not have to change.

<CardGroup cols={3}>
<Card title="Create an ECDSA Account" href="https://github.com/hashgraph/hedera-json-rpc-relay/blob/main/docs/examples/account-alias-ecdsa.js" img="/images/index/index-17.png" >
Create an ECDSA account that works with JSON-RPC to use EVM-compatible tooling, libraries, and wallets.
</Card>

<Card
  title="Set Up Metamask"
  href="https://github.com/hashgraph/hedera-json-rpc-relay/blob/main/docs/transfer-guide.md"
  img="/images/index/index-18.png"
>
  Add a Hedera network to Metamask using the [Hashio](http://hashio.io/)
  implementation of the JSON-RPC Relay. Sign transactions and queries to test
  your Hedera-powered application.
</Card>

<Card
  title="Libraries & Tooling"
  href="https://github.com/hashgraph/hedera-json-rpc-relay/tree/main/tools"
  img="/images/index/index-19.png"
>
  Integrate Hedera into Ethers, web3.js, Truffle, or Hardhat for building,
  debugging, and deploying your smart contract applications on the Hedera
  network.
</Card>

<Card title="JSON-RPC Relay" href="/core-concepts/smart-contracts/json-rpc-relay" img="/images/index/index-20.png">
JSON-RPC makes it easy for existing EVM-based libraries, dev tools, and wallets to work effortlessly on Hedera.
</Card>
</CardGroup>
